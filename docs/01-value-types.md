# Step 1: 值类型系统

## 这是什么？

Redis 使用 RESP（REdis Serialization Protocol）协议在客户端和服务器之间通信。
RESP 定义了 5 种可以在线路上传输的数据类型：

| RESP 前缀 | 类型 | 示例 | 说明 |
|-----------|------|------|------|
| `+` | 简单字符串 | `+OK\r\n` | 简短的、人类可读的字符串（不含换行） |
| `-` | 错误 | `-ERR unknown command\r\n` | 错误消息 |
| `:` | 整数 | `:1000\r\n` | 64 位有符号整数 |
| `$` | 批量字符串 | `$5\r\nhello\r\n` | 二进制安全的字符串（可包含任意字节） |
| `*` | 数组 | `*2\r\n$3\r\nfoo\r\n$3\r\nbar\r\n` | 有序的 RESP 值列表 |

这一步你要创建 Go 类型来表示这 5 种 RESP 类型。这些类型后续会用于编解码协议（Step 9-10）和构建命令响应。

## 为什么重要？

**Redis 中的每一份数据都通过这 5 种类型流动。** 当 `redis-cli` 发送 `SET mykey hello` 时，
服务器收到的是一个由批量字符串组成的 RESP 数组。当服务器回复 `+OK` 时，那就是一个简单字符串。
你的值类型是整个系统的"词汇表"。

## 设计决策

### 为什么用接口？

使用 Go `interface` 可以让 5 种类型存入同一个集合（比如数组）、通过同一个管道传递、
被通用代码处理。这对应 Redis 内部的 `robj` 类型——一个带类型标记的通用对象包装。

### 为什么用 `ValueType` 字节？

不用 `reflect` 或类型断言，每个值携带自己的类型码。更快，也更匹配 Redis 内部用字节标记类型的做法。

### null 的表示

批量字符串和数组可以是 "null"（和 "空" 不同）：
- Null 批量字符串：`$-1\r\n` — 表示"无值"（例如 GET 一个不存在的 key）
- 空批量字符串：`$0\r\n\r\n` — 表示"长度为零的字符串"
- Null 数组：`*-1\r\n` — 表示"没有数组"（例如 BLPOP 超时）
- 空数组：`*0\r\n` — 表示"零元素列表"

你的类型必须区分这两种情况。

## 真实世界的参考

在 Redis 源码（C 语言）中，对象系统位于 `object.c` 和 `sds.h`：
- `robj` 结构体包装了 `type` 字段（string/list/set/zset/hash）和 `encoding` 字段
- 对于线路协议，`sds`（Simple Dynamic String）同时处理简单字符串和批量字符串
- 在你的 Go 版本中，不需要 `robj` — 你的 `Value` 接口就是等价物

## 你要构建的

1. 一个 `ValueType` 类型和 5 个 RESP 类型常量
2. 一个 `Value` 接口，包含 `Type()` 和 `String()` 方法
3. 五个具体结构体：`StringVal`、`ErrorVal`、`IntVal`、`BulkVal`、`ArrayVal`
4. 每个类型的构造函数
5. Null 批量字符串和 Null 数组的特殊构造函数
